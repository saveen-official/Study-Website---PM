01 · concept

What is a pointer, really?

Every variable lives somewhere in memory. A pointer is just another variable — but instead of storing a number or a letter, it stores the address of where another variable lives.

Think of memory as a street of numbered houses. A normal variable is a house with something inside it (a value). A pointer is a piece of paper with a house's address written on it. The pointer doesn't contain the value — it tells you where to go to find the value.

int m = 25;
25
lives at 0x100000000A
int *p = &m;
0x100000000A
p "points to" m
The two symbols you need first

& (address-of) — put it in front of a variable to get its address: &m means "the address where m lives."

* (dereference / indirection) — put it in front of a pointer to get (or set) the value it points to: *p means "the value stored at the address p holds."

Declaring a pointer

syntax 
type *variable_name;

// Example: a pointer that can store the address of an int
int *ptr;

The * here doesn't mean "multiply" — in a declaration it means "this variable is a pointer." The type (e.g. int, float) must match the type of the variable it will point to.

A full first example

ptr1.c 
#include <stdio.h>

int main() {
    int m = 25;
    int *p = &m;          // p now holds the address of m

    printf("%d", *p);   // dereference p -> prints 25

    *p = 35;             // change m's value THROUGH p
    printf("%d", m);    // prints 35 - m really changed!

    return 0;
}

Nothing here is magic. *p = 35; walks to the address stored in p and overwrites whatever is there — and that "there" happens to be m. This is the entire idea behind "pass by reference" in C.

02 · concept

Functions that return more than one value

A normal return can only send back one value. But what if a function needs to hand back two or three results? That's what output parameters are for.

An output parameter is just a pointer passed into a function. The function dereferences it to write a result directly into the caller's variable — like leaving a note at an address the caller gave you.

Example: splitting a number into sign, whole part, and fraction

separate.c 
// num is the INPUT. signp, wholep, fracp are OUTPUTS (pointers)
void separate(double num, char *signp, int *wholep, double *fracp) {
    double magnitude;

    if (num < 0)
        *signp = '-';
    else if (num == 0)
        *signp = ' ';
    else
        *signp = '+';

    magnitude = fabs(num);
    *wholep = floor(magnitude);
    *fracp  = magnitude - *wholep;
}
Why the names end in "p"

signp, wholep, fracp — the trailing p is a naming convention meaning "this is a pointer." It's not required by C, but it makes code much easier to read because you instantly know which parameters are outputs (pointers) vs. which are plain inputs (copies).

Calling it

main() — calling separate() 
char sn;
int whl;
double fr;

separate(35.817, &sn, &whl, &fr);
// sn = '+', whl = 35, fr = 0.817

Notice: every output parameter is called with & — you're handing separate() the addresses of sn, whl, and fr so it can write results straight into them.

Output parameters vs. return values — when to use which

SituationUseWhy
Function produces exactly one result (e.g. a sum, an average) return Simple, and the value can be used directly in an expression
Function needs to produce two or more results (e.g. circumference and area) Output parameters (pointers) return can only carry one value back
Function just modifies existing variables (e.g. swap) Input/output parameters (pointers) No new value to "return" — you're changing what's already there

Matching pointer types — quick table

If the variable is......its address has type
char xchar * (pointer to char)
int xint * (pointer to int)
double xdouble * (pointer to double)
Type mismatch = bugs

If x is an int, its address must be stored in an int *, never a float * or double *. The pointer's type tells C how many bytes to read/write at that address.

03 · concept

Scope: where can a name be seen?

Scope is the region of your program where a name (a variable or function) is visible. Getting this right avoids the classic beginner error: "why can't I use this variable here?!"

Local scope

A variable declared inside a function (or inside { }) only exists there. Once the function ends, it's gone.

local scope 
void myFunction() {
    int localVar = 5;  // only visible inside myFunction
}
// localVar does not exist out here

Block scope

Even tighter: a variable declared inside { } — like inside an if or a loop — only exists within those braces.

block scope 
int main() {
    int x = 10;
    if (x > 5) {
        int y = 20;   // y only exists inside this { }
        printf("%d", y);
    }
    // printf("%d", y);  -> ERROR: y is not accessible here
}

Global scope

A variable declared outside every function lives for the whole program and can be read/changed by any function.

global scope 
int globalVar = 10;  // visible to ALL functions below

void myFunction() {
    printf("%d", globalVar);  // works fine
}
Rule of thumb

Declare variables in the smallest scope that works. Most variables should be local to the function (or even the loop) that uses them. Global variables are occasionally useful for true constants, but overusing them makes programs hard to debug because any function could be changing them.

Function names and scope

A function name has global scope by default — once declared, it can be called from anywhere below that point in the file. Marking a function static restricts it to only the file it's defined in, which is useful for "helper" functions you don't want other files to call.

static function 
static void helperFunction() {
    // This function can only be called within this file
}

04 · concept

Pointer arithmetic

When you "add 1" to a pointer, it doesn't move forward by 1 byte — it moves forward by the size of the type it points to.

arithmetic.c 
float x = 5.5;
float *p = &x;

printf("%p", p);    // e.g. 0x7ffee03bc678
p++;                  // move to the NEXT float in memory
printf("%p", p);    // 0x7ffee03bc67c  (moved by 4 bytes!)

A float takes up 4 bytes, so p++ jumps the address forward by 4 — not 1. If p were a double * (8 bytes), it would jump by 8. This is exactly why a pointer's type matters: it tells C how far "one step" is.

TypeTypical sizep++ moves forward by
char1 byte1 byte
int4 bytes4 bytes
float4 bytes4 bytes
double8 bytes8 bytes
Note on Lab 08, Q3

Your exact printed addresses will differ from anyone else's (and from run to run) — that's normal. What matters is that the difference between the "before" and "after" address matches the size of the data type (4 bytes for float).

05 · concept

Pointers and arrays are best friends

An array name is, deep down, just the address of its first element. This means you can walk through an array using either square brackets [ ] or pointer arithmetic — they're two views of the same thing.

arrays_and_pointers.c 
int a[5] = {10, 20, 30, 40, 50};
int *ptr = a;        // 'a' decays to &a[0] - no & needed!

printf("%d\n", *ptr);       // 10  (same as a[0])
printf("%d\n", *(ptr + 1));  // 20  (same as a[1])

2D arrays through pointers

A 2D array is "an array of arrays." arr[0][1] and *(*(arr + 0) + 1) are two ways of writing exactly the same thing.

ptr12.c 
int arr[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};

printf("%d", arr[0][1]);            // 2
printf("%d", *(*(arr + 0) + 1));   // 2 - same value!
You don't need to memorise the double-star form

For this course, the important takeaway is simply: arr[i][j] and pointer arithmetic give the same result, because under the hood an array is a contiguous block of memory addressed by its starting position. Stick with arr[i][j] in your own code — it's clearer.

Passing arrays to functions

When you pass an array to a function, you're really passing a pointer to its first element. That's why a function can modify the caller's array — and why you usually pass the array's length as a separate parameter (the function has no way to know how big the array is otherwise).

passing arrays (Lab 07, Q3 style) 
void findHottestColdest(int temps[], int size, int *hotIdx, int *coldIdx) {
    *hotIdx = *coldIdx = 0;
    for (int i = 1; i < size; i++) {
        if (temps[i] > temps[*hotIdx])  *hotIdx  = i;
        if (temps[i] < temps[*coldIdx]) *coldIdx = i;
    }
}

This combines two ideas from this guide: the array temps[] is passed by reference automatically, while hotIdx and coldIdx are explicit output parameters used to return two results.

Pointer to a pointer

A pointer can store the address of another pointer. You'll see this written as int **ptr2. It sounds scary but it's just one more layer of "the address of."

ptr11.c 
int var = 10;
int *ptr1 = &var;     // ptr1 holds the address of var
int **ptr2 = &ptr1;   // ptr2 holds the address of ptr1

printf("%d", var);     // 10
printf("%d", *ptr1);   // 10 - one dereference
printf("%d", **ptr2);  // 10 - two dereferences

0x...18

Each * "follows one arrow." *ptr2 follows the arrow from ptr2 to ptr1 (giving you an address). **ptr2 follows it twice, landing on var's value.

07 · concept

const pointers, void pointers, NULL

Pointer to a constant

const int *p means: "you cannot change the value through p" — but p itself can still be reassigned to point somewhere else.

ptr10.c 
int a = 10, b = 20;
const int *p = &a;

printf("%d", *p);  // 10
// *p = 15;       -> ERROR: cannot modify through a const pointer
p = &b;             // OK - p itself can be reassigned
printf("%d", *p);  // 20

NULL pointers

A pointer that doesn't point to anything yet should be set to NULL. Always initialise pointers — an "uninitialised" pointer could point to a random, dangerous address.

safe declaration 
int *ptr = NULL;

// Before using a pointer that might be NULL, always check:
if (fp == NULL) {
    printf("Error opening file.\n");
} else {
    // safe to use fp
}

void pointers (generic pointers)

A void * can point to any type, but you must "cast" it to the correct type before dereferencing it.

ptr6.c 
int a = 10;
double b = 3.0;
void *ptr;

ptr = &a;
printf("%d", *(int*)ptr);     // cast back to int* first

ptr = &b;
printf("%.2f", *(double*)ptr); // cast back to double*
Dangling pointers

After you free() dynamically allocated memory, the pointer still holds the old address — but that memory is no longer yours. Using it afterwards gives unpredictable results. Always set the pointer to NULL right after freeing it: ptr = NULL;

08 · concept

Function pointers (for Lab 08, Q5)

Just like a pointer can store the address of a variable, it can also store the address of a function. This lets you pass a function as an argument — so a "generic" function can decide how to compare things based on what you hand it.

function pointer syntax 
// A pointer to a function that takes (int, int) and returns int:
int (*compare)(int, int);

// max() accepts such a pointer as its third parameter:
int max(int a, int b, int (*compareFn)(int, int)) {
    if (compareFn(a, b) > 0)
        return a;
    return b;
}

Now you can write two different comparison functions and let the caller choose which logic max() should use:

two comparison strategies 
int compare(int a, int b) {
    if (a > b) return 1;
    if (b > a) return -1;
    return 0;
}

int compare_absolute(int a, int b) {
    return compare(abs(a), abs(b));
}

// In main():
max(8, -10, compare);            // -> 8   (normal comparison: 8 > -10)
max(8, -10, compare_absolute);   // -> -10 (|−10| > |8|, so -10 "wins")
Mental model

Pass a function's name (no parentheses, no arguments) to hand over its address. Inside max(), calling compareFn(a, b) actually runs whichever function was passed in. This is the same idea as a "strategy" — the logic for comparing is swapped out without rewriting max().

Worked example: Task 4 (Swap), fully traced

swap.c 
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 5, y = 10;
    printf("Before swap: a = %d, b = %d\n", x, y);

    swap(&x, &y);

    printf("After swap: a = %d, b = %d\n", x, y);
    return 0;
}
/* OUTPUT
Before swap: a = 5, b = 10
After swap: a = 10, b = 5
*/

Step by step, inside swap:

LineWhat happensa (=&x)b (=&y)temp
int temp = *a;copy the value at address a (which is x = 5) into temp5105
*a = *b;copy the value at address b (10) into the address a points to → x becomes 1010105
*b = temp;copy temp (5) into the address b points to → y becomes 51055

Because a and b are addresses of x and y in main(), every *a / *b write reaches all the way back into main()'s variables.

Worked example: Task 6 (Simplify a fraction)

The hint says to use the GCD (Greatest Common Divisor). Here's the structure — try filling in the GCD logic yourself before checking:

simplify_fraction.c — skeleton 
int gcd(int a, int b) {
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}

void simplify_fraction(int *numerator, int *denominator) {
    int g = gcd(*numerator, *denominator);
    *numerator   = *numerator   / g;
    *denominator = *denominator / g;
}

// main(): for input 8/12 -> gcd(8,12) = 4 -> 8/4=2, 12/4=3 -> "2/3"
Why is this "input/output"?

numerator and denominator start as the values typed by the user (input), and the same pointers are used to write the simplified result back (output) — overwriting the originals rather than creating new variables.

Common beginner mistakes

Forgetting & when calling

If a function expects int *x, you must call it with &myVariable, not just myVariable. Passing the value instead of the address is one of the most common compile errors for beginners (often shown as a type mismatch warning).

Confusing declaration * with dereference *

int *p; — the * here means "p is a pointer."
*p = 5; — the * here means "go to the address p holds, and write 5 there."
Same symbol, different meaning depending on context — this trips up almost everyone at first.

Using a pointer before it points anywhere

int *p; *p = 5; — p hasn't been told an address yet, so *p = 5 writes to a random/unsafe location. Always assign p = &something; (or p = NULL; if not ready yet) before dereferencing.

Mismatched pointer types

Don't store an int's address in a float *. The compiler will warn you — listen to that warning, because dereferencing will read/write the wrong number of bytes.

Glossary

pointer
A variable that stores the memory address of another variable.
& (address-of operator)
Returns the memory address of the variable it's applied to.
* (indirection / dereference operator)
In a declaration, marks a variable as a pointer. Elsewhere, accesses or sets the value at the address a pointer holds.
output parameter
A pointer parameter a function uses to "write back" a result into the caller's variable.
input/output parameter
A pointer parameter whose pointed-to value is both read from and written back to during the function call (e.g. swap, simplify_fraction).
scope
The region of code where a name (variable/function) is visible and usable.
pointer arithmetic
Adding/subtracting integers from a pointer moves it by that many elements (not bytes), based on the pointed-to type's size.
NULL
A special value meaning "this pointer points to nothing." Always check for it, and initialise pointers to it before use.
dangling pointer
A pointer that still holds an address after that memory has been freed — using it is unsafe.
void pointer
A "generic" pointer with no associated type; must be cast to a specific type before dereferencing.
function pointer
A pointer that stores the address of a function, allowing the function to be called indirectly or passed as an argument.
